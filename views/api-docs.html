Complete reference for Learning Notifier API endpoints, authentication, and usage examples.

Authentication Endpoints Examples Error Codes Interactive Docs

Authentication

Learning Notifier API uses admin code authentication for protected endpoints. Regular endpoints are public, but admin operations require a valid admin code.

POST /swagger-auth/verify

Verify admin code for Swagger UI access

Query Parameters:

Parameter Type Required Description
code string Yes Admin authentication code

Registration Endpoints

POST /register

Register a new user for daily learning notifications

Request Body:

Parameter Type Required Description
name string Yes User's full name
email string Yes User's email address
type string No Developer type (developer, frontend, backend, fullstack, student, other)
isAdmin boolean No Whether user is an admin
adminCode string If isAdmin=true Admin authentication code

Example Request:

{ "name": "John Doe", "email": "john@example.com", "type": "developer", "isAdmin": false }

Success Response (200):

{ "message": "Registration completed successfully", "user": { "_id": "64f1a2b3c4d5e6f7g8h9i0j1", "name": "John Doe", "email": "john@example.com", "type": "developer", "isAdmin": false } }
GET /register

Get all registered users (Admin only)

Request Body:

Parameter Type Required Description
adminCode string Yes Admin authentication code
GET /register/:id

Get a specific user by ID (Admin only)

Path Parameters:

Parameter Type Required Description
id string Yes User's MongoDB ObjectId
PUT /register/:id

Update a user (Admin only)

Request Body:

Parameter Type Required Description
name string No Updated user name
email string No Updated email address
type string No Updated developer type
adminCode string Yes Admin authentication code
DELETE /register/:id

Delete a user (Admin only)

Request Body:

Parameter Type Required Description
adminCode string Yes Admin authentication code

Notification Endpoints

POST /notify/test

Send a test notification to a specific user

Request Body:

Parameter Type Required Description
email string Yes Email address to send test notification
POST /notify/trigger

Trigger daily notifications for all users

Request Body:

No parameters required

System Endpoints

GET /app/health

Check API health status

Success Response (200):

{ "status": "ok", "message": "Learning Notifier API is running", "timestamp": "2024-01-15T10:30:00.000Z", "version": "1.0.0" }

Error Codes

Status Code Error Type Description
400 Bad Request Invalid request parameters or missing required fields
401 Unauthorized Invalid admin code or missing authentication
404 Not Found Resource not found (user, endpoint, etc.)
409 Conflict Resource already exists (duplicate email)
500 Internal Server Error Server-side error occurred

Example Error Response:

{ "statusCode": 400, "message": "User with this email already exists", "error": "Bad Request" }
